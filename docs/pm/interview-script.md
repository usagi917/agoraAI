# Agent AI — 経営コンサルタント向け AIシナリオ分析ツール：カスタマーインタビュースクリプト

---

## 1. インタビュー準備

### 対象者リクルーティング基準

**必須条件（Must）**
- 経営コンサルティングファーム（従業員50〜300名規模）に在籍
- 職位：マネージャー、シニアマネージャー、ディレクター、パートナーのいずれか
- 過去12ヶ月以内にシナリオ分析（市場分析、事業計画策定、M&Aデューデリジェンス等）を含むプロジェクトに関与した経験がある
- クライアントへのデリバリー責任を持つ立場である

**望ましい条件（Nice to have）**
- ChatGPT、Claude、Copilot等のAIツールを業務で試用した経験がある
- チーム（アナリスト・アソシエイト）のマネジメント経験がある
- 製造業、エネルギー、テクノロジーセクターのいずれかの案件経験がある

**除外条件**
- IT/デジタルコンサルティング専業（テクノロジーリテラシーが高すぎてバイアスがかかる）
- 独立コンサルタント（組織的な業務フローの課題が見えにくい）
- 直近で競合ツールの開発・販売に関与している人

### スクリーニング質問（5問）

1. **現在のご所属と職位を教えてください。**
   - 合格基準：50〜300名規模のコンサルファーム、マネージャー以上

2. **直近12ヶ月で、シナリオ分析や市場予測を含むプロジェクトに何件関与しましたか？**
   - 合格基準：2件以上

3. **そのプロジェクトで、分析作業にどの程度の時間を費やしましたか？（チーム全体の工数）**
   - 合格基準：具体的な数値を答えられる

4. **業務でChatGPT、Claude等の生成AIツールを使ったことはありますか？**
   - 合格基準：使用/未使用どちらでも可（セグメント分けに使用）

5. **45分のオンラインインタビューにご協力いただけますか？（録音の許可を含む）**
   - 合格基準：承諾

### インタビュー設定

| 項目 | 詳細 |
|------|------|
| 所要時間 | 45分（最大60分まで延長可） |
| 形式 | オンライン（Zoom/Google Meet）、対面も可 |
| 録音 | 開始前に明示的に許可を取得（録音同意書を事前送付） |
| 人数 | インタビュアー1名 + 記録係1名（記録係はカメラオフ推奨） |
| 録画 | 表情・ボディランゲージ観察のため録画も依頼（任意） |

### 必要な準備物

- [ ] 録音同意書（PDF、事前メールで送付）
- [ ] インタビュースクリプト（印刷またはサブディスプレイ表示用）
- [ ] サンプルレポート：EVバッテリー市場分析の出力例（PDF、画面共有用）
- [ ] 3Dグラフ可視化のスクリーンショット（3〜4枚）
- [ ] 記録シート（Notion/Googleスプレッドシート）
- [ ] タイマー（各セクションの時間管理用）
- [ ] 謝礼の案内文（インタビュー後に送付）

## 2. インタビュースクリプト（45分構成）

### ウォームアップ（5分）

#### 自己紹介テンプレート

> 「本日はお忙しいところお時間をいただきありがとうございます。私は[名前]と申しまして、[会社名]でプロダクトリサーチを担当しております。
>
> 今日は、コンサルティング業務におけるシナリオ分析や市場予測の進め方について、○○さんの実際のご経験をお聞きしたいと考えています。正解・不正解はありませんので、率直にお話しいただけると大変助かります。
>
> なお、本日のお話は録音させていただき、社内の分析目的のみに使用いたします。事前にお送りした同意書にご了承いただけましたでしょうか？
>
> 所要時間は45分程度を予定しています。途中でお答えになりたくないご質問があれば、遠慮なくおっしゃってください。」

#### アイスブレイク質問

- 「最近のプロジェクトはお忙しいですか？今どんな業界の案件が多いですか？」
- 「コンサルタントとしてのキャリアは何年目くらいですか？」

**注意：ここでは製品の説明を一切しない。「AIツールの調査です」とも言わない。「シナリオ分析業務の進め方について研究している」というフレーミングに留める。**

### コア探索：現在のワークフロー（15分）

#### 導入質問

> 「直近で、シナリオ分析や市場予測を含むプロジェクトについて教えていただけますか？守秘義務に触れない範囲で結構です。」

#### 深掘り質問

**プロジェクトの全体像**
1. 「そのプロジェクトでは、シナリオ分析はどの段階で、誰が行いましたか？」
2. 「分析にはどのくらいの期間がかかりましたか？チームは何名でしたか？」
3. 「最終的なアウトプットはどんな形式でしたか？（Excel、PowerPoint、レポート等）」

**ツールとプロセス**
4. 「データ収集から分析、レポーティングまでの流れを、ステップごとに教えていただけますか？」
5. 「Excelのモデルはどの程度の複雑さでしたか？シートは何枚くらい、変数はどのくらいありましたか？」
6. 「分析結果の可視化はどうされましたか？グラフやチャートはどうやって作りましたか？」

**ペインポイント探索**
7. 「そのプロジェクトで、一番時間がかかったのはどの部分でしたか？」
8. 「やり直しや手戻りが発生したことはありましたか？何が原因でしたか？」
9. 「クライアントからの『シナリオをもう一つ追加してほしい』という要望にはどう対応しましたか？」
10. 「アナリストが作った分析結果に対して、どの程度レビューや修正が必要でしたか？」

**理想状態**
11. 「もしその分析作業に制約が何もなかったら、理想的にはどうやりたかったですか？」
12. 「シナリオの数や分析の深さについて、『本当はここまでやりたかったけど時間的にできなかった』ということはありましたか？」

#### Mom Test チェックポイント
- 具体的なプロジェクト名（匿名可）と数字（期間、人数、工数）を引き出せているか？
- 「困っていますか？」ではなく「実際にどうしましたか？」と過去の行動を聞けているか？
- 相手が一般論ではなく自分の体験を語っているか？

---

### コア探索：AIツールの利用状況（10分）

#### 導入質問

> 「話は変わりますが、最近ChatGPTやClaude等の生成AIツールを業務で使われることはありますか？」

#### AIを使っている場合の深掘り

1. 「具体的にどんな場面で使っていますか？最近使った例を教えてください。」
2. 「AIの出力をそのままクライアントに見せたことはありますか？見せた場合、クライアントの反応はどうでしたか？」
3. 「AIの出力をどの程度編集・加工してから使いますか？」
4. 「AIを使って期待外れだった経験はありますか？何がダメでしたか？」
5. 「チーム内でAIの利用に関するルールやガイドラインはありますか？」
6. 「AIツールに月額でお金を払っていますか？個人で？会社で？」

#### AIを使っていない場合の深掘り

1. 「使っていない理由は何ですか？」
2. 「周りで使っている同僚はいますか？その人たちの評価はどうですか？」
3. 「もしAIを使うとしたら、どんな業務に使いたいですか？」
4. 「AIを業務に導入するとした場合、障壁になりそうなことは何ですか？」

#### 信頼性に関する質問

5. 「AIが出した市場予測や分析結果を、どの程度信頼できると思いますか？」
6. 「AIの分析結果に対して、クライアントはどう反応すると思いますか？」
7. 「『AIが分析しました』と言うことで信頼性が上がると思いますか、下がると思いますか？」

### デモ＆フィードバック（10分）

**重要：ここまでのセクションで十分な情報を得てからデモに移る。先にデモを見せると、それ以降の回答にバイアスがかかる。**

#### デモの進め方

> 「ここで少し、あるツールが出力したサンプルレポートをお見せしたいのですが、ご覧いただけますか？これはEVバッテリー市場の分析レポートのサンプルです。」

**手順：**
1. サンプルレポート（EVバッテリー市場分析）を画面共有する
2. **30秒間、何も説明せずに読んでもらう**（この間の表情・反応を記録係が観察）
3. 3Dグラフ可視化のスクリーンショットを見せる
4. **再び30秒間、何も説明せずに見てもらう**

#### フィードバック質問

1. 「最初に見た印象を教えてください。」（オープンエンド、誘導しない）
2. 「このレポートの中で、使えそうだと思った部分はどこですか？」
3. 「逆に、使えない・足りないと思った部分はどこですか？」
4. 「このレポートをそのままクライアントに見せられますか？見せられない場合、何を変える必要がありますか？」
5. 「普段ご自身のチームが作るレポートと比べて、どうですか？」
6. 「この3Dグラフはクライアントプレゼンで使えそうですか？」
7. 「このレベルの分析を自分のチームで作る場合、どのくらいの時間がかかりますか？」

#### 観察ポイント（記録係が記録）

- [ ] 最初にレポートのどの部分を見たか（目線の動き）
- [ ] 「おっ」「へえ」等の自発的な反応はあったか
- [ ] 前のめりになったか、腕を組んだか（ボディランゲージ）
- [ ] 質問が出たか（質問の内容＝関心のある領域）
- [ ] 否定的な反応が出た具体的な箇所

**注意：「すごいですね」「面白いですね」等の社交辞令を額面通りに受け取らない。具体的な行動意向で判断する。**

### WTP＆クロージング（5分）

#### 支払い意欲（WTP） — Van Westendorp法

1. 「このようなツールが月額制で利用できるとしたら、いくらくらいなら『安い、すぐ使ってみよう』と思いますか？」
2. 「いくらくらいだと『まあ妥当だな』と思いますか？」
3. 「いくらくらいだと『ちょっと高いけど、価値があれば払う』と思いますか？」
4. 「いくらを超えると『高すぎて無理』ですか？」

#### 導入意向

5. 「もし2週間の無料トライアルがあれば、試してみたいですか？」
   - 「はい」の場合：「いつ頃が都合よいですか？今抱えている案件で使えそうですか？」
   - 「いいえ」の場合：「何があれば試してみようと思いますか？」

6. 「社内で導入を決める場合、どなたの承認が必要ですか？どんなプロセスになりますか？」

#### 紹介依頼

7. 「同じような業務をされている方で、この話に興味を持ちそうな方はいらっしゃいますか？」

#### クロージング

> 「本日は貴重なお話をありがとうございました。大変参考になりました。後日、お礼のメールをお送りいたします。追加でお聞きしたいことが出てきた場合、ご連絡させていただいてもよろしいでしょうか？」

## 4. インタビュー分析フレームワーク

### 回答のコーディング方法

| コード | カテゴリ | 説明 |
|--------|----------|------|
| P-TIME | ペイン：時間 | 分析作業に時間がかかりすぎる |
| P-QUAL | ペイン：品質 | 分析の精度・深さが不十分 |
| P-SCAL | ペイン：拡張性 | シナリオ追加・変更への対応が困難 |
| P-COLB | ペイン：協業 | チーム間の連携・レビューに課題 |
| P-VIZZ | ペイン：可視化 | グラフ・チャート作成に手間がかかる |
| A-TRUST | AI：信頼 | AI出力への信頼度に関する言及 |
| A-USE | AI：利用 | 現在のAI利用状況 |
| A-BLOCK | AI：障壁 | AI導入の障壁に関する言及 |
| D-POS | デモ：肯定 | デモへの肯定的反応 |
| D-NEG | デモ：否定 | デモへの否定的反応 |
| D-MISS | デモ：不足 | デモで足りない要素の指摘 |
| W-PRICE | WTP：価格 | 具体的な価格への言及 |
| W-TRIAL | WTP：トライアル | トライアル意向に関する言及 |
| U-CASE | ユースケース | 具体的なユースケースの言及 |

### Go / No-Go 判定基準（10名インタビュー完了後）

| 判定項目 | Go基準 | Pivot基準 | Kill基準 |
|----------|--------|-----------|----------|
| シナリオ分析のペインが存在する | 8名以上 | 5〜7名 | 4名以下 |
| AI分析結果を意思決定に使いたい | 6名以上が肯定 | 3〜5名 | 2名以下 |
| 既存ワークフローからの切替意向 | 5名以上 | 3〜4名 | 2名以下 |
| ChatGPT直接利用より価値がある | 6名以上 | 3〜5名 | 2名以下 |
| WTP月額5万円以上 | 5名以上 | 3〜4名 | 2名以下 |
| トライアル意向 | 7名以上 | 4〜6名 | 3名以下 |
| デモへの肯定的反応 | 7名以上 | 4〜6名 | 3名以下 |

**総合判定：**
- **Go**：7項目中5項目以上がGo基準
- **Pivot**：Go基準3〜4つ、かつKill基準0
- **Kill**：Kill基準2つ以上

### 謝礼の相場

| 職位 | 謝礼金額 | 形式 |
|------|----------|------|
| マネージャー | 5,000〜10,000円 | Amazonギフト券 |
| シニアマネージャー / ディレクター | 10,000〜15,000円 | Amazonギフト券 or スタバカード |
| パートナー | 15,000〜30,000円 | Amazonギフト券 or 銀行振込 |

**補足：** パートナー層は謝礼よりも「知的好奇心」「インサイト共有」の方が響くことが多い。
