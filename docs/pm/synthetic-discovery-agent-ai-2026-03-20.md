## 1. 現在のプロダクト要約

### 現在のプロダクトが提供しているもの

Agent AI は、見た目以上に技術的実体のある分析ワークベンチです。ユーザーはプロンプトを入れ、必要なら文書を添付し、テンプレートと実行プロファイルを選ぶと、世界モデル構築、シナリオ探索、PM評価を含むライブ実行を行えます。結果画面では、レポート、シナリオ比較、追加質問、グラフ履歴、PM評価を確認できます。

リポジトリ上で明確に見える主要な利用面は次の3つです。

1. ビジネス分析
2. シナリオ探索
3. 政策シミュレーション

加えて、`PM Board` は差別化機能として存在しますが、現状では主UIの入口よりもAPIや結果モデル側で目立っています。

### このプロダクトをもっとも使いそうな人

リポジトリの事実から見る限り、想定ユーザーはカジュアル利用者ではありません。もっとも相性が良いのは、次のような人たちです。

- 文書量の多い仕事をしている
- 不確実性の高いテーマを構造化してまとめる必要がある
- 複数シナリオや複数視点の比較を重視する
- 単純な ChatGPT 利用より重い導線でも、出力が再利用できるなら許容する

現在のコードと内部資料から見て、もっとも有力な顧客仮説は次の3つです。

- 中堅の日本語圏戦略コンサルティングマネージャー
- 事業会社の経営企画 / 新規事業マネージャー
- 新規事業責任者 / 主任PM

なお、リポジトリ内の内部PM文書はコンサルタントを先行セグメントに寄せています。これは実証済みではありませんが、プロダクトの形とは整合しています。

### 価値が出る瞬間

このプロダクトで価値が出る瞬間は、3Dグラフそのものではありません。価値が出るのは、ユーザーが次の状態に到達できた時です。

- 雑多な入力から、使える初稿を一発で得られる
- 複数シナリオを手作業で縫い合わせずに比較できる
- 追加質問にゼロからやり直さず答えられる
- 上司、役員、クライアントに説明できる程度の根拠や追跡性を持てる

つまり、価値の中核は「シナリオを伴うメモやレビュー初稿の最初の60-70%を圧縮できるか」にあります。

### 導入障壁になりそうな点

- セットアップがまだオープンソースの試験導入ツール寄りで、完成済み業務アプリには見えにくい
- 外部LLM呼び出し前提のため、機密文書との相性に強い懸念がある
- UIとREADMEの語りが技術中心で、ユーザー成果中心になり切っていない
- エクスポート、認証、ワークスペース、共有、PowerPoint / Docs 連携がなく、業務フローに埋め込みにくい
- 根拠は意識されているが、引用を前面に出した成果物化がまだ弱い

### 継続利用の障壁になりそうな点

- 出力をそのまま上位者に見せるには、まだ手直しが必要そう
- プロジェクト蓄積、チームワークスペース、バージョン比較、共有レビュー導線がなく、再利用性が弱い
- 表面上は「すごいが重い」印象になりやすく、日常業務に常駐しにくい
- `PM Board` は差別化要素だが、週次の軽い意思決定ツールとしてはまだ見えにくい

### 過剰な点と不足している点

**初回導入に対して過剰な点**

- 認知アーキテクチャの深さを前面に出している
- 3Dグラフが中心の価値のように見える
- 実行モードと実行プロファイルが多く、最初の理解を難しくしている
- BDI / ToM / multi-Colony の技術語彙がトップレベルの説明に出すぎている

**継続利用に対して不足している点**

- 根拠付きエクスポート
- 認証とワークスペースの基礎
- 共有 / レビュー / 版管理
- 低摩擦の導入導線
- 買い手向けのパッケージングと投資対効果の説明

#### 観察された課題

- 文書の構造化と論点整理が最大の工数
- パートナーレビューや追加シナリオで手戻りが多い
- AI出力は根拠追跡性が弱いと実務に乗らない

#### 需要シグナル

- 60点以上の初稿を短時間で出せるなら試したい
- 追加質問とシナリオ再実行には明確な価値を感じている
- 根拠付きならパートナーレビュー前工程として使える可能性がある

#### 乗り換え障壁

- クライアント機密
- PowerPoint / クライアント向けメモへの再利用性
- 既存のアナリスト作業フローへの慣れ

#### 支払い意思または投入可能なリソース

- まずはサニタイズ案件での小規模パイロット

|---|---|---|---|---|
| 価値 | より良い初稿が出るなら、ChatGPT + アナリストから乗り換える価値がある | 出力が粗く、手直しで価値が消える | 低-中 | 同一文書セットでの成果物比較 |
| 価値 | シナリオ比較と追加質問Q&A は、グラフ可視化より重要 | ユーザーが「面白いデモ」としか見ない | 中 | グラフを隠した簡素版デモで反応比較 |
| 使いやすさ | 買い手は `pipeline` / `profile` / `template` の複雑さを許容する | どのモードを選ぶか分からず離脱する | 低 | 単一導線UIでのタスク完了テスト |
| 使いやすさ | PMチームは `PM Board` が前面に出れば使う | PM用途では依然として重すぎる | 低-中 | 軽量なPMレビュー試作を3人に見せる |
| 事業性 | セルフホスティング + APIキーでも有償パイロットに進める | 試用摩擦が高すぎて反復利用に到達しない | 低 | 案内付きホステッド型パイロットとの比較 |
| 事業性 | 日本語ファーストの位置づけは初期差別化になる | 買い手が Speeda や ChatGPT に流れる | 中 | 3種類の訴求メッセージ比較 |
| 実現性 | 根拠先行のエクスポートは全面改修なしで追加できる | 引用の網羅率や整形品質が安定しない | 中 | 固定の書き出し経路を1本作り、根拠のない主張の比率を見る |
| 実現性 | 大きなグラフと詳細ビューは残してもパイロットを邪魔しない | 読込、複雑さ、描画負荷が導入率を下げる | 中 | 初期表示では隠し、必要時のみ遅延読込するパイロット版を比較 |

## 8. 最終提案

この順位付けは `prioritize-features` の考え方に沿っています。軸は、影響度、工数、リスク、戦略整合性です。

### 優先度テーブル

| 順位 | 改善案 | 期待インパクト | コスト | 優先度 |
|---|---|---|---|---|
| 1 | 根拠付きメモ出力パック | 非常に高い | 中 | P0 |
| 2 | 先行セグメント向け導線の単純化 | 非常に高い | 中 | P0 |
| 3 | ホステッド型パイロット導線とコスト制御 | 高い | 中 | P1 |
| 4 | ワークスペース / 認証 / 版管理 / 共有の基礎 | 高い | 高 | P1 |
| 5 | メインUI上の軽量なPMレビュー | 中-高 | 中 | P2 |

### 1. 根拠付きメモ出力パック

- 何を変えるか:
  - 主要主張ごとに根拠参照を紐づける
  - クライアント向け / 役員向けメモの固定フォーマットを用意する
  - Markdown と PowerPoint または Docs に落とせる書き出しを追加する
- どの顧客課題に効くか:
  - 「AI出力はそのまま見せられない」
  - 「結局スライドに手で起こし直す」
  - 「どの情報に基づくのか説明しにくい」
- 期待インパクト:
  - 試用から実務利用への転換率を最も押し上げる
  - 手直し工数を減らし、レビュー時の信頼を上げる
- 実装または運用コスト:
  - 中
- 優先度:
  - `P0`
- まず何を検証すべきか:
  - 想定ターゲット3名に現行出力と引用付き書き出し試作を見せ、使える下書きになるまでの編集時間を比較する

### 2. 先行セグメント向け導線の単純化

- 何を変えるか:
  - `pipeline`, `preview`, `quality` など内部都合の言葉を主UIから後退させる
  - 入口を「戦略メモ初稿」「シナリオ比較」「事業アイデアレビュー」のようなジョブ別導線に変える
  - 3D表示 / 認知ビューは任意表示にし、初回価値をレポート中心に寄せる
- どの顧客課題に効くか:
  - 「何を選べばよいか分からない」
  - 「技術デモに見えて、買う理由が伝わらない」
- 期待インパクト:
  - 初回理解と起動率の改善
  - ChatGPT との差分を買い手の言葉で説明しやすくなる
- 実装または運用コスト:
  - 中
- 優先度:
  - `P0`
- まず何を検証すべきか:
  - 5人前後の対象候補に新旧画面を見せ、「何のための製品か」を自分の言葉で説明できるか比較する

### 3. ホステッド型パイロット導線とコスト制御

- 何を変えるか:
  - セルフホスティング前提だけでなく、パイロット用のホステッド環境または案内付き環境を作る
  - 実行前に時間とコストの目安を出す
  - 失敗しにくい既定プロファイルと利用ガイドを用意する
- どの顧客課題に効くか:
  - 「試すまでが重い」
  - 「APIキーや運用が面倒」
  - 「実行コストが読めない」
- 期待インパクト:
  - 仮説探索インタビューからパイロットへの移行率改善
  - 技術に強くない買い手でも試しやすくなる
- 実装または運用コスト:
  - 中
- 優先度:
  - `P1`
- まず何を検証すべきか:
  - セルフホスト導線とホステッド / 案内付きパイロット導線のどちらが試用意向を高めるか確認する

### 4. ワークスペース / 認証 / 版管理 / 共有の基礎

- 何を変えるか:
  - ユーザー認証
  - プロジェクトワークスペース
  - 保存済み実行履歴
  - 再実行比較
  - 社内共有 / レビューリンク
- どの顧客課題に効くか:
  - 「チームで扱えない」
  - 「前回の分析が再利用しにくい」
  - 「権限や共有が曖昧で社内導入できない」
- 期待インパクト:
  - 継続利用率とチーム導入率の改善
  - 有償パイロットの業務導入準備度向上
- 実装または運用コスト:
  - 高
- 優先度:
  - `P1`
- まず何を検証すべきか:
  - パイロット候補に対して、これが導入の必須条件か、導入後の要件かを聞き、開発順を確定する

### 5. メインUI上の軽量なPMレビュー

- 何を変えるか:
  - `PM Board` をメインUIから直接起動できるようにする
  - グラフ前提ではなく、前提マップ / 検証計画 / 推奨事項を前面に出す
  - 10分以内の軽量レビュー導線を設計する
- どの顧客課題に効くか:
  - 「事業アイデアレビューをもっと厳しく、構造化して進めたい」
  - 「重い分析ではなく、次の検証計画がほしい」
- 期待インパクト:
  - 周辺セグメントでの差別化強化
  - 現在見えにくい価値の可視化
- 実装または運用コスト:
  - 中
- 優先度:
  - `P2`
- まず何を検証すべきか:
  - PM / 新規事業責任者3名程度に軽量試作を見せ、ChatGPT との差分が体感できるか確認する

### 今は後回しにすべきこと

- 3Dグラフをさらに派手にすること
- 認知アーキテクチャの説明を増やすこと
- 実行モードをさらに増やすこと
- 最初の導線が固まる前の広いセグメント拡張

## 9. 今すぐやるべき1つの実ユーザー検証

もし次に1つだけ実ユーザー検証をやるなら、これです。

**直近で完了したサニタイズ済み案件を使い、戦略コンサルのマネージャー1名と成果物の並列比較テストを行う。**

進め方:

1. 直近案件の元資料と、最終的に提出したメモまたはスライド資料を用意する
2. 同じ入力で Agent AI を回す
3. マネージャーに次の2つを渡す
   - いつもの手作業 / ChatGPT補助の下書き作成フロー
   - 引用付きメモ出力試作に整えた Agent AI 出力
4. 30-45分で「実際に上へ見せる水準」に編集してもらう
5. 次を測る
   - 使える下書きになるまでの時間
   - 根拠のない主張をどれだけ削除したか
   - 来週も使いたいか
   - 何が阻害要因だったか

この1本が優先な理由:

- もっとも重要な価値仮説を直接テストできる
- 追跡可能性とエクスポートが本当に阻害要因か見える
- 一般論のインタビューより、はるかに強い証拠が得られる

## 10. 結論

このリポジトリには、すでに相当量の技術的プロダクトが入っています。今回の `synthetic discovery` の結論は、「価値がない」ではありません。もっと狭い結論です。

- 価値仮説は十分にある
- 最有力の切り込み先は、日本語圏の戦略チームであって、全員ではない
- 売り方は技術アーキテクチャより、再利用可能な意思決定下書きに寄せるべき
- いま足りないのは、さらなる認知深度ではなく、信頼、書き出し、ワークフロー適合、反復可能なチーム利用

繰り返しますが、これは `synthetic discovery` であり、実ユーザー検証ではありません。次の5-8件の実インタビューと、1-2件の成果物パイロットを設計するための地図として扱うべきです。
