## 1. Summary

この文書は、Agent AI を「単発のシミュレーションツール」から「1万人のエージェントが住む常設のデジタル社会」へ拡張するための要件をまとめたものです。

最初の版では、1万人を全員フル LLM で動かすことはしません。普段は住民プロフィールを DB に持ち、必要なときだけ一部を選び、少人数の深い会議に上げる階層方式を採用します。

## 3. Background

### 現在あるもの

今の Agent AI には、次の強い土台があります。

- 文書から世界モデルを作る機能
- BDI 認知アーキテクチャ
- 3層メモリ
- Theory of Mind
- Game Master によるラウンド制実行
- Swarm による多視点実行
- PM Board による少人数の専門人格会議

### 今まだ足りないもの

今の実装は、シミュレーションごとにエージェントを生成して終わる形です。1万人の住民が常に存在し、過去の経験や社会関係を持ちながら暮らしている状態ではありません。

また、1万人全員に毎回 LLM を呼ぶのは、速度、コスト、安定性の面で現実的ではありません。

### なぜ今やるか

この構想は、今の資産を壊さずに拡張できます。特に以下の3点がつながります。

- 住民全体を持つ DB 層
- 必要な住民だけを選ぶ選抜層
- 選ばれた少人数を既存 BDI / Game Master / PM Board で深く動かす実行層

## 4. Objective

### 目的

ユーザーがあるテーマを投入したときに、1万人規模の仮想社会から関係の深い住民を選び、必要に応じて専門人格も追加し、意思決定、市場予測、行動予測に使える集団知を返せるようにする。

### 何が良い状態か

- エージェントがシミュレーション間で消えない
- 住民ごとに個性、立場、記憶、社会関係がある
- テーマごとに関係の深い住民だけを選べる
- 少人数の深い議論を既存の認知機構で回せる
- 出力の良し悪しを見た目ではなく評価指標で判断できる

### Key Results

1. 1万件の住民プロフィールを DB に生成し、再利用できる
2. テーマ投入後 5 秒以内に候補住民 200 件を選べる
3. 候補住民から 20-50 体の軽量推論と 5-10 体の深い推論を実行できる
4. 1回の標準実行を 15 分以内で完了できる
5. 既存ベースラインより良い予測精度または判断支援品質を示せる

## 5. Market Segment(s)

この機能の初期ユーザーは、次のような「不確実な未来を多視点で読みたい人」です。

### Segment A: 経営企画 / 新規事業

- 何を知りたいか: 新市場に参入したらどう動くか
- 欲しいもの: 利害関係者の反応、採用障壁、勝ち筋

### Segment B: 政策・公共領域

- 何を知りたいか: 制度変更に対して社会がどう反応するか
- 欲しいもの: 支持層、反対層、二次影響

### Segment C: 研究・実験用途

- 何を知りたいか: エージェント社会の振る舞いと予測精度
- 欲しいもの: 再現実験、比較評価、論文化しやすい設計

## 6. Value Proposition(s)

### ユーザーにとっての価値

- 単一の AI の答えではなく、社会の分布として見られる
- 似た人格だけでなく、対立する人格も混ぜて議論できる
- 必要な専門人格を追加できる
- 結果が毎回一発芸ではなく、住民の記憶と関係が次回に引き継がれる

### 既存手段との差

- ChatGPT に直接聞く方法より、誰の視点かが明確
- 普通の ABM より、必要な場面だけ深い言語推論を使える
- 単発のマルチエージェント実験より、住民社会が持続する

## 7. Solution

### 7.1 UX / User Flow

ユーザーフローは次の形にする。

1. ユーザーがテーマを入力する
2. ユーザーが必要なら「追加したい人格」を指定する
3. システムが住民 DB から関係の深い住民を選ぶ
4. システムが軽量な集団反応を作る
5. システムが代表住民と専門人格だけを深い会議に上げる
6. システムが最終的な分布、論点、予測、推奨アクションを返す

### 7.2 Key Features

#### Feature 1: Agent Population Registry

1万人の住民を保存する台帳を持つ。

各住民は最低でも次を持つ。

- 基本属性
- 性格特性
- 価値観
- 専門領域
- 信念ベクトル
- 社会ネットワーク
- 記憶サマリ
- 過去の重要行動

受け入れ条件:

- 1万件の住民データを生成して保存できる
- 住民はシミュレーション後も消えない
- 同じ住民を複数回の会議で再利用できる

#### Feature 2: Population Generator

住民を統計的に生成する仕組みを持つ。

初期版では LLM を必須にしない。分布ベースで生成し、必要なら後で一部だけ詳細人格化する。

受け入れ条件:

- 生成時に属性分布を設定できる
- 特定の職業、年代、地域に偏らないよう制御できる
- 後から住民を追加生成できる

#### Feature 3: Agent Selection Engine

テーマに対して関係の深い住民を選ぶ。

選抜は単純な類似度だけではなく、偏りを避けるための多様性制約を入れる。

選抜条件の例:

- テーマとの関連度
- 専門性
- 立場の違い
- ネットワーク中心性
- 少数派の保持

受け入れ条件:

- テーマごとに候補住民 200 件を返せる
- 選抜理由を説明できる
- 賛成派だけに偏らない

#### Feature 4: Hierarchical Simulation Engine

住民を3段階で処理する。

- Layer 1: 200-500 体の簡易反応
- Layer 2: 20-50 体の軽量 LLM 推論
- Layer 3: 5-10 体の深い BDI 推論

受け入れ条件:

- 各段階の入力と出力が定義されている
- 深い推論に上がる条件が明確
- 実行コストを段階ごとに計測できる

#### Feature 5: Expert Persona Assembly

ユーザーが「経済学者を追加したい」「環境活動家を追加したい」と指定したときに、住民 DB から選ぶか、新しく専門人格を作って会議に入れられる。

受け入れ条件:

- 追加人格の役割を指定できる
- 既存住民から選ぶか新規生成かを区別できる
- 追加人格が会議ログに明示される

#### Feature 6: Persistent Social Graph

住民同士の関係を永続化する。

関係の例:

- 友人
- 同僚
- 競争相手
- 支持者
- 影響元

受け入れ条件:

- 関係が住民間で保存される
- 会議やイベント後に関係強度が更新される
- ある住民が誰に影響されやすいか追跡できる

#### Feature 7: Evaluation Harness

この機能が本当に役に立つかを評価する仕組みを持つ。

最低限の評価軸:

- 予測精度
- 判断支援品質
- 立場の多様性
- キャリブレーション
- 再現性

受け入れ条件:

- 同じテーマでベースライン比較ができる
- どの層が効いたかを分析できる
- 実験結果を保存できる

### 7.3 Technology

### 既存資産の活用方針

- `world_builder` はテーマ理解と状況整理に使う
- `agent_generator` は恒久住民生成ではなく、会議用の動的エージェント化に使う
- `Game Master` と `CognitiveAgent` は Layer 3 の深い議論に使う
- `PM Board` は Expert Persona Assembly の原型として使う

### 新しく必要な技術要素

- 住民プロフィール用テーブル
- 社会関係グラフ用テーブル
- テーマと住民の類似度検索
- 代表住民抽出ロジック
- 評価実験の保存基盤

### 7.4 Assumptions

- 初期版では 1万人全員をフル LLM で動かさない
- 初期版では住民プロフィールは統計生成でよい
- 初期版では「社会全体の常時稼働」より「テーマ投入型」を優先する
- 初期版では住民の人格精度より、選抜と評価の仕組みを優先する

### Out of Scope

初期版では次をやらない。

- 1万人全員のリアルタイム常時推論
- 完全自律で永遠に動く社会
- 現実世界を完全再現することの保証
- 高精度な政治予測や株価予測の断定

## 8. Release

### Release 1: Foundation

期間の目安: 4-6 週間

入れるもの:

- Agent Population Registry
- Population Generator
- Agent Selection Engine
- 最低限の評価基盤

出さないもの:

- 深い BDI 会議
- 専門人格の追加 UI
- 常時稼働

### Release 2: Deliberation

期間の目安: 4-6 週間

入れるもの:

- Hierarchical Simulation Engine
- Layer 3 への BDI 接続
- Expert Persona Assembly
- 会議ログ表示

### Release 3: Persistent Society

期間の目安: 6-8 週間

入れるもの:

- Persistent Social Graph
- 記憶更新
- 住民状態の時系列追跡
- より強い評価実験

## MVP Definition

最初の MVP は「1万人の社会を完成させること」ではない。

最初の MVP は次の問いに答えること。

「住民台帳 + 選抜 + 少人数の深い会議、という階層方式は、単一 LLM や単純な少人数会議より良いか」

これに答えられれば、次のフェーズへ進む価値がある。
